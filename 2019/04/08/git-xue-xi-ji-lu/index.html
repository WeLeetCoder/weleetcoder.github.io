Git 学习

https://git-scm.com/book/zh/v2

配置 user 信息

在使用 Git 的时候，需要先配置用户信息，配置用户名和邮箱。

$ git config --global user.name "your_name"    
$ git config --global user.email "your_email@example.com"

注意: user.name=”your_name” 这种格式是错误的，用空格分开即可

上面的 global 是全局设置，能设置为：

optionsdescription
--local配置指针对某个仓库有效
--global配置对当前用户的所有仓库有效
--system配置对于系统所有登录的用户都有效

查看 config 的配置，可以加上 --list。

$ git config --list --local # 产看当前仓库的配置
$ git config --list --global 
$ git config --list --system

还可以为 Git 的命令添加别名：

$ git config --local alias.s status        # 为 git status 配置了别名 git s
$ git s
On branch master
nothing to commit, working tree clean

开始学习

创建一个 Git 仓库

​ 在学习 Git 之前，首先，我们需要创建一个 Git 仓库，才能够使用 Git 管理我们的文件，使用下列命令创建一个 Git 仓库：

$ git init
Initialized empty Git repository in C:/Users/ZAKI/Code/gitlearn/.git/

创建完成了 Git 仓库，我们可以来尝试下 git 有关的命令。

当 global 的配置和 local 的配置冲突的时候，会使用哪个呢？
$ git config --local user.name "gitlearn"

$ git config --local --list
core.repositoryformatversion=0
core.filemode=false
core.bare=false
core.logallrefupdates=true
core.symlinks=false
core.ignorecase=true
user.name=gitlearn

$ git config --global --list
user.email=zq2640710087@gmail.com
user.name=zhongqi

$ touch a.txt               # 先创建一个 a.txt

$ git add a.txt             # 提交 a.txt 到暂存区，先这样

$ git status                # 使用 git status 查看仓库当前状态
On branch master

No commits yet

Untracked files:
  (use "git add <file>..." to include in what will be committed)

        a.txt

nothing added to commit but untracked files present (use "git add" to track)

$ git commit -m "add a.txt" # -m 参数是指提交 msg

$ git status                # 提交完成之后再查看
On branch master
nothing to commit, working tree clean

$ git log                   # 查看提交日志
commit 80195406471d1be5c0301ef6d26cb2b161bdd972 (HEAD -> master)
Author: gitlearn <zq2640710087@gmail.com>
Date:   Mon Apr 8 10:07:02 2019 +0800

    add a.txt

在最后，可以看到Author 是 gitlearn，是我们使用git config --local 命令配置的，可见，这三个配置的优先级是local > global > system。

了解工作目录、暂存区、版本仓库

我们在写代码的目录，即工作目录，而使用了git add <filename> 提交了之后，这个文件存储的地方就是暂存区，而版本仓库是通过 git commit -m “msg“ 提交的。
