配置环境

如果有 ruby 环境的小伙伴可以这样：

$ gem install sass

在命令行中运行 Sass：

sass input.css output.css

gem 是 ruby 带的一个类似于包管理器的命令。sass命令

没有 ruby 环境也可以写，不过需要 node 环境，本文中使用的就是 nodejs 环境，使用 gulp 前端构建工具，也能够轻松的将 sass 转换为css。

安装 nodejs

这步就不用说了吧。。。

安装 gulp

使用 npm install -g gulp 全局安装 gulp 前端构建工具。

新建一个项目目录：

$ mkdir sass-learn && cd sass-learn

进入根目录下：

$ npm init

这个命令会生成一个 package.json 文件。

然后再在项目目录中安装 gulp 开发依赖。

$ npm install gulp --save-dev

或者：

$ npm i -D gulp

当然，只是安装了 gulp 还不行，还需要安装 sass 的编译器：

$ npm i -D gulp-sass

等待它安装完成即可。

$ npm i -D del

这个是用于清理先前打包好的 css 文件的。

然后是在项目目录下新建一个 gulpfile.js 文件。

在里面输入如下代码：

var gulp = require('gulp');
var sass = require('gulp-sass');
var del = require('del');

// gulp.task 是用于定义命令，与命令如何执行的 API。
// gulp.src 是指定目录下，哪些文件需要被操作。
// gulp.dest 是将通过处理的文件输出到哪个目录。

// 定义了名字叫 clean 的任务，用于清理编译好的 css 文件。
gulp.task('clean', function() {
  return del.sync(["./src/css/*.css"]);
})

// 定义了一个名字叫 sass 的任务，用于执行 sass 编译。
gulp.task("sass", function() {
  return gulp
    .src(["./src/sass/**/*.scss", "./src/sass/**/*.sass"])
    // outputStyle 是可以选的： nested | expanded | compact | compressed
    .pipe(sass({ outputStyle: 'compressed' }).on("error", sass.logError)) 
    .pipe(gulp.dest("./src/css"));
});

// 定义了一个名字叫 sass:w 的任务，用于监控 src/sass 目录下的所有 .scss\.sass 后缀名的文件的改动，有改动就自动执行重新编译 sass 命令
gulp.task('sass:w', function() {
  gulp.watch(
    ["./src/sass/**/*.scss", "./src/sass/**/*.sass"],
    gulp.series("sass")
  );
});

// 这个是默认任务，直接在项目目录下执行 gulp 就是执行这个任务，执行这个任务会先执行两个任务，分别是 clean、sass
gulp.task("default", gulp.series("clean", "sass"));

上面的代码是定义了构建任务。

outputStyle 属性可用如下值：

  • nested 是 Sass 的默认输出，很有结构层次感，清晰的反映了 css 与 html 的对应结构。
  • expanded 这个呢则是更像是人写的一样，我们一会改成这个就好了。
  • compact 这个比上面两种写法节省空间，不过还是一个样式独占一行。
  • compressed 这个就是压缩了，删除所有无意义的空格，注释等等无用代码，将文件压缩至最小。

再创建一个 src 目录，在 src 目录下再创建一个 sass 目录。

现在的目录中有如下目录

.
|-- gulpfile.js                  gulp 任务文件
|-- node_modules                 node 包管理目录
|-- package-lock.json            包依赖版本锁定文件 
|-- package.json                 包依赖文件
`-- src                          我们在这下面写代码
    `-- sass                     在这里写sass代码

接下来在 sass 文件中创建一个 test.scss 文件。

test.scss
body {
    div {
        .test {
            font: {
                size: 24px;
                weight: 600;
            }
            color: red;
        }
    }
}

然后执行 gulp sass 就会执行在 gulpfile.js 中定义名字叫 sass 的任务。

$ gulp sass
[09:41:53] Using gulpfile ~\sass-learn\gulpfile.js
[09:41:53] Starting 'sass'...
[09:41:53] Finished 'sass' after 53 ms

命令完成后，我们可以看到在 src 目录下生产了一个css目录，里面有一个 test.css 。

test.css
body div .test{font-size:24px;font-weight:600;color:red}

css 就生成完毕了。

也可以执行：

$ gulp sass:w

这个命令也是在 gulpfile.js 中定义的，直接运行就好了，它会监听文件变化，并且重新编译 sass 文件。

好啦，到这里环境的配置也就结束了，下面是正式学习 Sass 语法的时候了。

SassScript

SassScript 提供了节省 CSS 代码的写法，可以用更少的代码写出更丰富的 CSS，并且还支持使用变量，运算等等额外功能。

先学下下面可能会用到的指令

以@ 符号开头的就是指令了，先学几个常用的指令，指令非常有用，对于复杂的 SassScript 我们经常使用指令。

  • @debug 用于将SassScript表达式的值输出到标准错误流中，其实就是相当于将值打印出来输出到命令行而已，这个指令非常有用，对于定位错误。
  • @warn 和 @debug 指令差不过，不过有两点不一样：
    1. 通过命令行指定 –quiet 可以不输出警告
    2. 警告会输出那个文件，第几行报了警告。
  • @error 会产生一个 fatal 错误，输出错误的追踪信息。并且终止执行SassScript。

以上的是用于打印信息的指令，下面可能会用到。

变量声明

在 SassScript 中用的最多的就是 变量了，变量必须以美元符号开头，赋值的方法是和 CSS 的属性的写法是一样的。

$width: 10rem;

然后可以直接使用变量：

div {
   width: $width; 
}

Sass 的变量和编程一样，也有作用域，在代码块里面定义的变量只能在代码块内使用，在外面是无法访问的。在最外层定义的变量可以在全局都可以访问，也就是全局变量。如果需要将局部变量转换为全局变量可以添加 !global 声明：

#main {
    $width: 5em !global;
    width: $width;
}

div {
    width: $width;
}

编译为：

#main {
  width: 5em;
}

div {
  width: 5em;
}

数据类型

编程语言有数据类型，SassScript 也有数据类型，SassScript 支持一下 6 种数据类型：

  • 数字：1, 2, 3, 10px 10rem ...
  • 字符串，有引号的和无引号的字符串：str1,'str2',"string3"
  • 颜色值：red, #e2e2e2, rgb(255, 0, 0), rgba(0, 0, 0, .5) ...
  • 空值：null
  • 数组，用空格或者逗号隔开：1rem 1em 12px ... OR success, error, warning, ...
  • maps，也就是键值对：(key1: val1, key2: val2 ...)
  • function
  • true and false

SassScript 支持CSS的属性值，比如 Unicode 字符集，或者 !important 声明。Sass 不会处理这些值，这些值会被视为无引号的字符串。

数值与字符串

@mixin content($el, $content, $w) {
    .#{$el} {
        content: $content;
        width: $w;
    }
}
div {
    @include content(warning, "WARNING", 20px);
    @include content(success, "SUCCESS", 10rem);
}

会被编译为：

div .warning {
  content: "WARNING";
  width: 20px;
}

div .success {
  content: "SUCCESS";
  width: 10rem;
}

数组

数组指的是 Sass 如何处理 CSS margin: 10px 15px 0 0 或者 font-face: Helvetica, Arial, sans-serif 这样通过空格或者逗号分隔的值。独立的值也可以被视为一个数组（只包含1个值的数组）。

Sass 数组本身没有太多的功能，但是 Sass 有内置的方法，为数组添加了更多的功能：

  • nth 函数可以直接访问数组中的某一项；
  • join 函数可以将多个数组连接在一起；
  • append 函数可以在数组中添加新值；

可以用@each 指令遍历数组中的每一项，获取每项数组的值。

数组也可以包含子数组：

$ar: (
    (1px 2px 3px 4px #666)
    (2px 3px 4px 5px #eee)
);

@each $item in $ar {
    .bx#{index($ar, $item)} {
        box-shadow: $item;
    }
}

会被编译为：

.bx1 {
  box-shadow: 1px 2px 3px 4px #666;
}

.bx2 {
  box-shadow: 2px 3px 4px 5px #eee;
}

上面的代码中：

$ar 中包含了子数组，遍历这个数组，使用 @each 指令，并且在内部使用了 index 函数，这个是 Sass 内置的函数，可以获取值在数组内的索引。数组可以使用括号括起来，就想上面那样，也可以像这样：

$ar: 1px 2px, 5px 6px;

可以用逗号分隔两个子数组，也可以使用空格分隔，不过需要套上括号：

$ar: (1px 2px) (3px 4px);

数组被编译为 CSS 后，Sass 不会添加或者保留圆括号。所以：

(1px 2px) (5px 6px) 和CSS中的 1px 2px, 3px 4px 是一样的。

maps

maps 也就是相当于 一个JavaScript对象而已，像这样：

$map: (
    (
        width: 10px,
    ),
    (
        width: 20px,
    )
);

maps 也是可以被遍历的，可以使用 @each ：

$map: (
    (
        width: 10px,
        height: 20px,
        background: green,
        color: white
    ),
    (
        width: 20px,
        height: 30px,
        background: red,
    )
);

@for $i from 1 through length($map) {
    .mp#{$i} {
        @each $attr, $val in nth($list: $map, $n: $i) {
            #{$attr}: $val;
        }
    }
}

会被编译为：

.mp1 {
  width: 10px;
  height: 20px;
  background: green;
  color: white;
}

.mp2 {
  width: 20px;
  height: 30px;
  background: red;
}

数字

数字可以带单位：

$width: 10px;
$font-size: 1.3rem;
$height: 20px * 4; // 80px
$w: 20px * 20px; // 400px*px Error: 400px*px isn't a valid CSS value.

注意：有单位的不能和有单位的一起做乘法运算，做除法运算的话单位会抵消，不同单位也不能一起做运算。

字符串

字符串分为带引号的和不带引号的，例如：

.alert-info {
    content: "警告！";
    font-family: system-ui, -apple-system, Segoe UI;
}

颜色

颜色值也是可以用来运算的：

$base-color: #0000fc;
.alert-info {
  color: $base-color + #666;
}

.alert-info {
  color: #6666ff;
}

布尔值

布尔值可以通过比较运算符（ < <= > >= != == ）来得到结果：

$bg-mode: dark;
.bg{
       @if $bg-mode == dark { // true
        background-color: black;
    } else {
        background-color: light;
    }
}

除了上述的运算符以外，Sass 还支持 and or not 3个逻辑运算。

$alert-grade: 3;
$alert-type: low;
.alert-info {
  @if  $alert-grade > 2 and $alert-type == high {
    color: red;
  } @else if $alert-grade < 3 or $alert-type == low {
    color: blue;
  } @else if not ($alert-type != low){
    color: gray;
  }
}

表达式

  • 变量
  • 方法调用(nth($list,1))
  • 特殊的函数
  • 父选择器符（&）

注释

Sass 支持两种注释，但是其实算是有3种，还有一种是特殊的。

两种的话是：多行注释和单行注释，三种的话就要加一种强制注释：

// 单行注释

/*
 * 多行注释
 */

/*!
 * 强制注释
 */

单行注释是不会出现在 css 文件中的，多行注释是会原样出现在 css 文件中，但是不会出现在压缩的css文件中，而强制注释会原样存在。

在注释中也是可以使用 interpolation 表达式的。就像这样：

$version: '1.1.0';
/*! version: #{ $version } */

其实算起来还有一种注释，叫文档注释，也就是可以被 SassDoc 读取的注释，注释被解析为 markdown，然后被用作文档：

/// Computes an exponent.
///
/// @param {number} $base
///   The number to multiply by itself.
/// @param {integer (unitless)} $exponent
///   The number of `$base`s to multiply together.
/// @return {number} `$base` to the power of `$exponent`.
@function pow($base, $exponent) {
  $result: 1;
  @for $_ from 1 through $exponent {
    $result: $result * $base;
  }
  @return $result;
}

通常是用来描述函数，或者 mixin 的作用的。

插值（interpolation）

插值几乎可以使用在 SassScript 的任何地方：

  • 选择器样式
  • 属性名声明
  • 用户自定义属性值
  • css 规则
  • @extends
  • 普通的css @imports
  • 有引号或无引号字符串
  • 特殊的函数
  • 普通css函数名
  • 注释中
@mixin corner-icon($name, $top-or-bottom, $left-or-right) {
  .icon-#{$name} {
    background-image: url("/icons/#{$name}.svg");
    position: absolute;
    #{$top-or-bottom}: 0;
    #{$left-or-right}: 0;
  }
}

@include corner-icon("mail", top, left);

@mixin inline-animation($duration) {
  $name: inline-#{unique-id()};

  @keyframes #{$name} {
    @content;
  }

  animation-name: $name;
  animation-duration: $duration;
  animation-iteration-count: infinite;
}

.pulse {
  @include inline-animation(2s) {
    from { background-color: yellow }
    to { background-color: red }
  }
}

去除引号变为无引号的字符串：

.example {
  unquoted: #{"string"}; // 使用插值后会去除有引号字符串的引号
}

比如，当使用 calc 函数的时候，当这样调用的时候，可能需要保持 20px - 10px 所以要用 interpolation 语法。

.wid {
  width: call(get-function('calc', true), #{'20px - 10px'});
}
用户自定义属性
$primary: #81899b;
$accent: #302e24;
$warn: #dfa612;

:root {
  --primary: #{$primary};
  --accent: #{$accent};
  --warn: #{$warn};

  // Even though this looks like a Sass variable, it's valid CSS so it's not
  // evaluated.
  --consumed-by-js: $primary;
}

父选择器

.alert {
  // The parent selector can be used to add pseudo-classes to the outer
  // selector.
  &:hover {
    font-weight: bold;
  }

  // It can also be used to style the outer selector in a certain context, such
  // as a body set to use a right-to-left language.
  [dir=rtl] & {
    margin-left: 0;
    margin-right: 10px;
  }

  // You can even use it as an argument to pseudo-class selectors.
  :not(&) {
    opacity: 0.8;
  }
}
添加前缀
.accordion {
  max-width: 600px;
  margin: 4rem auto;
  width: 90%;
  font-family: "Raleway", sans-serif;
  background: #f4f4f4;

  &__copy {
    display: none;
    padding: 1rem 1.5rem 2rem 1.5rem;
    color: gray;
    line-height: 1.6;
    font-size: 14px;
    font-weight: 500;

    &--open {
      display: block;
    }
  }
}

可以使用 & 代表父级选择器名称，使用了 & 以后，Sass 处理这个选择器的时候就不会加上父级选择器了。也就是这样：

.nav {
  background-color: #036;
  body.dark & {
    color: red;
  }
}

.nav {
  background-color: #036;
}

body.dark .nav {
  color: red;
}

可见，父级选择器并没有出现在 body.dark 的前面。

占位选择器

%alert {
    width: 100%;
    padding: 10px;
    border-radius: 4px;
}

.alert-info {
    @extend %alert;
    background: blue;
}

.alert-warning {
    @extend %alert;
    background: yellow;
}

占位选择器并不会被输出到 css 文件中去：

.alert-info, .alert-warning {
  width: 100%;
  padding: 10px;
  border-radius: 4px;
}

.alert-info {
  background: blue;
}

.alert-warning {
  background: yellow;
}

常用指令

流程控制指令

if($condition, $if-true, $if-false)

这是一个内置的方法，但是也是用来控制表达式输出的，如果 $condition 是真，则返回 $if-true 假则返回 $if-false。

@if
@if ... {
    // do something 1
} @else if ... {
    // do something 2
} @else {
    // do something 3
}

$theme: dark;
.theme {
    @if $theme == light {
        background-color: white;
    } @else if $theme == dark {
        background-color: black;
    } @else {
        background: grey;
    }
}

.theme {
  background-color: black;
}

@mixin avatar($size, $circle: false) {
  width: $size;
  height: $size;

  @if $circle {
    border-radius: $size / 2;
  }
}

.square-av { @include avatar(100px, $circle: false); }
.circle-av { @include avatar(100px, $circle: true); }
@each
@each <variable ...> in <expression> { ... }

$sizes: 40px, 50px, 80px;

@each $size in $sizes {
  .icon-#{$size} {
    font-size: $size;
    height: $size;
    width: $size;
  }
}
使用 @each 遍历 maps
$icons: ("eye": "\f112", "start": "\f12e", "stop": "\f12f");

@each $name, $glyph in $icons {
  .icon-#{$name}:before {
    display: inline-block;
    font-family: "Icon Font";
    content: $glyph;
  }
}

@each 还可以解构，如果需要获取 lists 中的 list 的值，可以使用这种语法。

$icons:
  "eye" "\f112" 12px,
  "start" "\f12e" 16px,
  "stop" "\f12f" 10px;

@each $name, $glyph, $size in $icons {
  .icon-#{$name}:before {
    display: inline-block;
    font-family: "Icon Font";
    content: $glyph;
    font-size: $size;
  }
}
@for
@for <variable> from <expression> to <expression> { ... }

@for <variable> from <expression> through <expression> { ... }

@for $i from 1 through 3 {
  ul:nth-child(3n + #{$i}) {
    background-color: lighten($base-color, $i * 5%);
  }
}
@while
@while <expression> { ... }

@while 指令如果表达式的值为真，则继续运行代码块的代码，如果表达式的的值为假，则停止运行。

@function scale-below($value, $base, $ratio: 1.618) {
  @while $value > $base {
    $value: $value / $ratio;
  }
  @return $value;
}

$normal-font-size: 16px;
sup {
  font-size: scale-below(20px, 16px);
}

在 Sass 中，只有两种值被视为假值：false null 。

混合指令

除了流程控制指令以外，我们用的更多的可能就是 @mixin 指令了。

@mixin

mixin 就像是一块定义好的样式块，可以通过传入参数来决定输出怎么样的样式，也就是一块能够根据参数来复用的样式块。

@mixin <name> { ... }

@mixin <name>(<arguments ...>) { ... }

@mixin reset-list {
  margin: 0;
  padding: 0;
  list-style: none;
}

@mixin horizontal-list {
  @include reset-list;

  li {
    display: inline-block;
    margin: {
      left: -2px;
      right: 2em;
    }
  }
}

nav ul {
  @include horizontal-list;
}

任意多个参数传递：

@mixin order($height, $selectors...) {
  @for $i from 0 to length($selectors) {
    #{nth($selectors, $i + 1)} {
      position: absolute;
      height: $height;
      margin-top: $i * $height;
    }
  }
}

@include order(150px, "input.name", "input.address", "input.zip");

接收任意多个关键字参数，对于任意多个关键字参数可以使用 keywords 函数来将它变成 map。

@mixin syntax-colors($args...) {
  @debug keywords($args); // (string: #080, comment: #800, $variable: $60b)

  @each $name, $color in keywords($args) {
    pre span.stx-#{$name} {
      color: $color;
    }
  }
}

@include syntax-colors(
  $string: #080,
  $comment: #800,
  $variable: #60b,
)

Sass也支持像 ES6 一样的参数传递方式，用 ... 将数组作为参数值传递：

$form-selectors: "input.name", "input.address", "input.zip" !default;

@include order(150px, $form-selectors...);

除了传递参数以外，还可以在 mixin 中定义内容块，可以在调用的时候注入代码到 mixin 中去：

@mixin hover {
  &:not([disabled]):hover {
    @content;
  }
}

.button {
  border: 1px solid black;
  @include hover {
    border-width: 2px;
  }
}

.button {
  border: 1px solid black;
}

.button:not([disabled]):hover {
  border-width: 2px;
}
@include

使用 @include 才能调用 mixin。

@include <name>;
@include <name>(<arguments ...>);

自定义函数

我们可以通过自定义函数来完成一些事情，比如计算平法：

@function pow($base, $exponent) {
  $result: 1;
  @for $_ from 1 through $exponent {
    $result: $result * $base;
  }
  @return $result;
}

.sidebar {
  float: left;
  margin-left: pow(4, 3) * 1px;
}

函数的参数基本和 mixin 的参数是一致的，但是函数是有返回值的，所以。

@return

函数的返回值就使用 @return 来返回。

样式继承

样式的继承，可以通过 @extend 指令来完成：

@extend
.error {
  border: 1px #666;
  background: #fdd;
  &-info {
    @extend .error;
    border-width: 3px;
    border-radius: 4px;
  }
}

.error, .error-info {
  border: 1px #666;
  background: #fdd;
}

.error-info {
  border-width: 3px;
  border-radius: 4px;
}

@extend 指令会把所有继承的相关的都继承下来像这样：

.error {
  border: 1px #666;
  background: #fdd;
  &:hover {
    color: red;
  }
  &-info {
    @extend .error;
    border-width: 3px;
    border-radius: 4px;
  }
}

.error, .error-info {
  border: 1px #666;
  background: #fdd;
}

.error:hover, .error-info:hover {
  color: red;
}

.error-info {
  border-width: 3px;
  border-radius: 4px;
}

上面的的例子中，.error-info 扩展了 .error ，还把它的 hover 也扩展了过来，只要和 .error 相关的类，都将被 .error-info 扩展。

下面是一个比较复杂的例子：

.content nav.sidebar {
  @extend .info;
}

// This won't be extended, because `p` is incompatible with `nav`.
// 这个不会被扩展，因为 p 和 nav 是不兼容的。也就是不匹配。 
p.info {
  background-color: #dee9fc;
}

// There's no way to know whether `<div class="guide">` will be inside or
// outside `<div class="content">`, so Sass generates both to be safe.
// 这个呢则是因为无法知道 guide这个类是在 .content 的里面还是外面，所以，Sass 会把两个类都生成
.guide .info {
  border: 1px solid rgba(#000, 0.8);
  border-radius: 2px;
}

// Sass knows that every element matching "main.content" also matches ".content"
// and avoids generating unnecessary interleaved selectors.
// Sass 知道之后元素匹配了 main 之后才匹配 .content 并且会避免生成不必要的交错选择器
main.content .info {
  font-size: 0.8em;
}

在扩展某个类的时候，这个类不存在，将会要求创建这个类，并且报错，如果要忽略这个报错，可以添加 !optional 声明达到这个目的：

a.important {
    @extend .notice !optional;
}
在媒体查询上扩展

在指令中使用 @extend 时（比如 @media 中）有一些限制：Sass不可以将 @media 外的CSS规则样式延伸给指令内层的 CSS，这样会生成大料无用的代码，也就是说在 @media 或者其他的 CSS 指令中@extend ，必须延伸给相同指令层中的选择器。

例如下面的例子：

@media print {
  .error {
    border: 1px #f00;
    background-color: #fdd;
  }
  .seriousError {
    @extend .error;
    border-width: 3px;
  }
}

上面这个例子是可以的，因为 .error 在 @media 的内部，但是下面这种就不可以了：

.error {
  border: 1px #f00;
  background-color: #fdd;
}

@media print {
  .seriousError {
    // INVALID EXTEND: .error is used outside of the "@media print" directive
    @extend .error;
    border-width: 3px;
  }
}

@at-root

这个指令会把所有在内部的类都移动到最外面去。

body {
  div {
    nav {
      width: 10px;
      @at-root .header {
        height: 20px;
      }
    }
  }
}

body div nav {
  width: 10px;
}

.header {
  height: 20px;
}
@content

使用 @content 可以向 @mixin 中导入内容:

 @mixin apply-to-ie6-only {
  * html {
    @content;
  }
}
@include apply-to-ie6-only {
  #logo {
    background-image: url(/logo.gif);
  }
}

* html #logo {
  background-image: url(/logo.gif);
}

变量的作用域：

$color: white;
@mixin colors($color: blue) {
  background-color: $color;
  @content;

.colors {
  background-color: blue;
