    <input type="submit" value="Upload Image" name="submit">
</form>

</body>
</html>

文件操作

打开、读取、写入、刷新缓冲区

打开文件的模式：r w a x r+ w+ a+ x+

r 是读取，w是写入，a是追加，x是创建新文件以只写模式。

w 如果文件不存在则创建，文件存在则清空文件再写，以 w 模式打开的文件，文件指针是在文件的开头。

以 a 模式打开的文件文件指针是在文件末尾。

x 模式是创建一个新的文件，如果文件已经存在了则返回 FALSE 并报错。

带加号的都是同时能读能写的。

$file = fopen('xxx.txt', 'r') or die ('Unable to open');

面向对象编写 （OOP）

OOP 编程的话，主要是用 class 将细节封装，提供相应的api供使用者使用，达到使用者符合开发者的编程逻辑的目的，以防使用的过程中出现意外错误。提高可复用性

class Person {
    // ...
}

上述代码中定义了一个 Person 类，这就是最简单的类定义了。这个类什么都没有做。

类定义了要使用，可以使用 new 来创建类的实例。

$person = new Person();

类成员有属性和方法两种。

class Person {
    public $name = 'zhongqi';
    public function eat($food) { echo 'eat ' . $food; }
}

上面的例子中 $name 就是属性，而 eat 则是一个方法，类内部的函数叫做方法，而不是函数。

类有 3 种修饰符：

  • public 无论在类内部还是外部都能访问
  • protected 在内的内部和子类的内部能够访问，主要使用与需要被继承的类
  • private 只能在当前类内部访问，子类和外部都是无法访问的

提供了 3 种修饰符，主要是为了对类更好的进行封装。被 public 修饰的属性或者方法无论是在类的实例，还是类的内部方法中，或者子类中都是可以访问的，而被 protected 修饰的属性或者方法在类的实例中是无法访问的，private 修饰的则只有类的内部才能够访问。这三种方式的话，private 则通常用于不需要外部知道的细节使用，做到隐藏内部实现细节。protected 的话，通常用于要继承的类中。

类的继承

class Person {
    private $name = 'zhongqi';
    public function eat($food) { echo 'eat ' . $food; }
}
class Man extends Person {
    public $sex = 'male';
    public function run() { echo 'run'; }
}
$man = new Man();
$man->eat('bread'); // eat bread

abstract class Shap {
    public function __construct($color = 'black') {
        $this->color = $color;
    }
    abstract public function getArea();   // 抽象方法必须被子类实现，如果没有实现，将无法被实例化
}

class Square extends Shap {

}

class Circle extends Shap {

}

$shape = new Circle('red');
var_dump($shape->color)

抽象类不能够被直接实例化，必须被其他的类继承，才能够使用。

为了提高封装性，我们通常为内部的属性设置 get 和 set 方法，通过 get 方法获取内部属性，通过 set 方法来设置内部属性。
